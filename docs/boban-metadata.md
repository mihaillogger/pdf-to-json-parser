## 1. Что конкретно от меня требуется (чек-лист)

1. **Поиск DOI** — первое, что делаю. Регулярка по тексту (приоритет — титульная страница).
2. **Коннектор к CrossRef API** — обогащение метаданных по DOI. **Опциональный, отключается флагом CLI.**
3. **Локальная LLM-фоллбэк** — если DOI/CrossRef не сработали: скармливаю текст 1-й страницы и достаю `title`, `authors`, `abstract`.
4. **OCR-фоллбэк (Tesseract)** — для страниц без текстового слоя; режим **двуязычный `eng+rus`**.
5. **Нормализация авторов** — строго формат `"Фамилия, Имя"`.
6. **Логика `metadata_source` и `metadata_confidence`** — проставляю источник и уверенность [0..1].
7. **Оффлайн-режим** — обязателен. Всё должно работать без интернета (флаг `--offline`).
8. **README по моей части** — модель и провайдер LLM, описание оффлайн-режима, **стоимость обработки одного документа**.
9. **Передать Мишгану**: в схему `Metadata` добавить недостающее поле `normative` (см. §6).

> ⚠️ **КАТЕГОРИЧЕСКИ НЕЛЬЗЯ** брать `title` из имени файла (`Статья 31.pdf` и т.п.). Только из текста титульного листа + обогащение через CrossRef. Имя файла — это просто id документа, оно ненадёжно (см. §5).

## 2. Контракт выхода (что я отдаю оркестратору Мишгана)

На выходе мой модуль возвращает **готовый объект `Metadata`** из `schemas.py`, а не сырой `dict`.

```python
from parser.schemas import Metadata, PageBlock

def extract_metadata(
    blocks: list[PageBlock],  # блоки от Матвея (extractor.get_page_blocks)
    raw_text: str,
    *,
    use_crossref: bool = True,
    use_llm: bool = True,
    offline: bool = False,
) -> Metadata:
    ...
```

### ⚠️ ФОРМАT ВХОДА (ЗАФИКСИРОВАНО — сверено с main)

`get_page_blocks()` отдаёт **`list[PageBlock]`, где `PageBlock` — это Pydantic-объект**
из `parser.schemas`, **НЕ словарь**. Доступ только по атрибутам (`b.text`, `b.bbox.top`).

| Поле | Тип | Важно |
|------|-----|-------|
| `text` | `str \| None` | **`None` у картинок!** |
| `font_size` | `float \| None` | **`None` у картинок** |
| `bbox` | `BBox` | объект: `b.bbox.left/top/right/bottom` |
| `page_number` | `int` | 1-индексация |
| `block_type` | `str` | `"text"` или `"image"` |
| `is_bold` | `bool` | жирный шрифт — сигнал заголовка |

Правила работы с блоками (чтобы не словить ошибки на `None`):
- **Перед текстом фильтровать**: `[b for b in blocks if b.block_type == "text" and b.text]`.
- Доступ **по атрибутам**, не по ключам: `b.bbox.top`, а не `b["bbox"]["top"]`.
- **Свой `PageBlock`/`BBoxDict` не объявлять** — импортировать из `schemas`.
- Текст в блоках **уже очищен** (`clean_text`): переносы склеены, пробелы схлопнуты.
- Колонтитулы (шапка/футер) Матвей **уже срезал** по координатам.
- `title` — самый крупный `font_size` на стр. 1; `is_bold=True` усиливает уверенность.

Поля `Metadata` (раздел 4.2 ТЗ), которые на мне:

| Поле | Тип | Обяз. | Правило заполнения |
|------|-----|-------|--------------------|
| `title` | `str` | да | Из титульного листа / CrossRef. Никогда из имени файла. |
| `title_en` | `str \| null` | нет | Перевод, если оригинал не на английском. Иначе `None`. |
| `authors` | `list[str]` | да | Формат `"Фамилия, Имя"`. Пустой список `[]`, если не извлечены. |
| `abstract` | `str \| null` | да | Аннотация единым текстом. `None`, если в документе нет. |
| `keywords` | `list[str]` | да | Пустой список `[]` при отсутствии. |
| `doi` | `str \| null` | да | Канонический вид `10.1016/j.ces.2025.121219`. `None`, если не найден. |
| `journal` | `str \| null` | нет | Лучше всего тянется из CrossRef. |
| `year` | `int \| null` | нет | 4 цифры. |
| `metadata_source` | `str` | да | `"pdf"` / `"crossref"` / `"manual"` / `"hybrid"`. |
| `metadata_confidence` | `float [0..1]` | да | См. §4. |
| `normative` | `... \| null` | — | **Пока нет в схеме!** См. §6. По эталону = `null`. |

> Очистку текста (`strip`, висячие переносы `"поверхно-\nстный"`) делать руками не нужно — она зашита в `BaseSchema`. Я отдаю «грязный» текст, схема нормализует.

## 3. Пайплайн извлечения метаданных (порядок и фоллбэки)

```
1. DOI: регулярка по raw_text (приоритет — стр. 1)
        паттерн: 10\.\d{4,9}/[-._;()/:A-Za-z0-9]+   (затем нормализация: lower, без хвостовой пунктуации)
            │
            ├── DOI найден ──► CrossRef API (если use_crossref и not offline)
            │                      ├── 200 OK ──► заполняем title/authors/journal/year
            │                      │             metadata_source = "crossref", confidence ≈ 0.9
            │                      └── ошибка/таймаут ──► фоллбэк ниже
            │
            └── DOI не найден ─┐
                               ▼
2. Локальная LLM (если use_llm): текст 1-й страницы → промпт
   "Извлеки title, authors, abstract строго в JSON".
   metadata_source = "pdf" или "hybrid", confidence ниже (≈ 0.5–0.7)
                               │
                               ▼
3. Чисто эвристики/regex по тексту (всегда доступно, в т.ч. offline):
   title = самый крупный шрифт сверху 1-й стр. (font_size от Матвея),
   authors/abstract = по ключевым якорям ("Abstract", аффилиации).
   metadata_source = "pdf", confidence низкий (≈ 0.3–0.5)
```

Если страница без текстового слоя → перед шагом 1 прогнать **OCR (Tesseract, `eng+rus`)**, чтобы получить текст.

Всегда: что не извлекли — `None` или `[]` (никаких отсутствующих ключей), факт фиксируем в логе и понижаем `metadata_confidence`.

---

## 4. Логика `metadata_source` / `metadata_confidence`

| Сценарий | `metadata_source` | `metadata_confidence` (ориентир) |
|----------|-------------------|----------------------------------|
| DOI найден + CrossRef ответил | `"crossref"` | 0.9 |
| Часть из CrossRef, часть из PDF/LLM | `"hybrid"` | 0.7–0.85 |
| Только PDF/LLM, без CrossRef | `"pdf"` | 0.4–0.7 |
| Ничего надёжного не вытащили | `"pdf"` | ≤ 0.3 |

> Эталон: `metadata_source: "crossref"`, `metadata_confidence: 0.9`. То есть удачный CrossRef → смело высокий confidence.

---

## 5. Подводные камни корпуса (моя зона)

- **Кириллические имена файлов** (`Статья 31.pdf`, `Статья 62.pdf`):
  - если внутри русский текст (аффилиации, литература) → OCR в режиме `eng+rus`;
  - если это просто переименованные английские статьи → **всё равно** тянуть метаданные из текста, не из имени файла.
- **Разорванный титульник**: метаданные часто порваны переносами строк и колонками. CrossRef по DOI даёт чистый результат — но DOI сперва нужно надёжно найти.
- **Supplementary / Supporting Information**: документ может включать приложение (с ~11-й стр.). Метаданные берём с **основного** титульного листа, не с титула приложения.
- **Формат авторов строгий** — `"Фамилия, Имя"` (`"Yao, Yunjin"`), а не `"Yunjin Yao"`. Нормализация обязательна.

## 6. КРИТИЧНО: расхождение схемы с эталонным JSON

В реальном эталонном JSON (Приложение А ТЗ) в объекте `metadata` есть поле, которого **нет** в текущей `src/parser/schemas.py`:

```json
"metadata": {
  ...
  "metadata_confidence": 0.9,
  "normative": null        ← ЭТОГО ПОЛЯ НЕТ В СХЕМЕ
}
```

**Действие:** сказать Мишгану добавить в `class Metadata` поле с дефолтом, иначе итоговый JSON может не пройти автоматическую валидацию проверяющих:

```python
normative: Any | None = Field(None, description="Поле из эталонного JSON, по умолчанию null")
```

Аналогичные «скрытые» поля проверяющих есть и в `Section` (`status`, `status_effective_from`) — это уже зона Ромчика/Мишгана, но если увижу падение валидации по метаданным — флажу первым делом `normative`.

## 7. Требования из ТЗ к LLM/внешним API (п. 6.6–6.7) — на мне

1. CrossRef/OpenAlex — **опционально и отключаемо флагом** (`--no-crossref` или общий `--offline`).
2. LLM — допускается для вспомогательных задач (нормализация авторов, разбор титульника), **но обязательно**:
   - (а) указать модель и провайдера в README;
   - (б) предусмотреть **оффлайн-режим**;
   - (в) зафиксировать **стоимость обработки одного документа** (для локальной LLM — это близко к нулю по деньгам, но фиксируем время/токены).
3. Запрещён hard-coding под конкретные журналы корпуса.

## 8. Предлагаемые CLI-флаги (согласовать с Мишганом)

```
--offline            полностью без сети (CrossRef выключен, только локально)
--no-crossref        не ходить в CrossRef, но LLM/эвристики разрешены
--no-llm             без LLM, только regex/эвристики + (опц.) CrossRef
--llm-model NAME     какую локальную модель использовать
```

## 9. Зависимости (ставить ТОЛЬКО через `uv add`)

- `requests` или `httpx` — CrossRef API
- `pytesseract` + системный `tesseract` (с языковыми пакетами `eng`, `rus`) — OCR
- локальная LLM-обвязка (уточнить с командой: `llama-cpp-python` / `ollama`-клиент / `transformers`)

> Системный `tesseract` — это внешняя зависимость уровня ОС, её надо отдельно прописать в README (раздел «установка»): `brew install tesseract tesseract-lang` (macOS) / `apt install tesseract-ocr tesseract-ocr-rus` (Linux).
